3) XSD (mini) - Que significa "validar"

XML "valido" significa: el XML esta bien formado y ademas cumple las reglas del XSD.

En este curso, el XML enlaza el esquema con xsi:noNamespaceSchemaLocation. Asi, herramientas como VS Code pueden validar mas facil. 


<biblioteca version="1.0"
            xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
            xsi:noNamespaceSchemaLocation="biblioteca.xsd">
  ...
</biblioteca>

Nota: este HTML te sirve para XPath. La validacion XSD la haces en VS Code (o con herramientas externas).

4) Ejercicios guiados (para clase)

Haz estos ejercicios con el laboratorio de XPath (arriba).

  1. Selecciona todos los libros: //libro
  2. Selecciona solo titulos: //libro/titulo
  3. Filtra por genero Fantasia: //libro[genero='Fantasia']
  4. Filtra por paginas > 200: //libro[paginas > 200]/titulo
  5. Cuenta prestamos abiertos: count(//prestamo[estado='ABIERTO'])
  6. Suma precios: sum(//libro/precio)

Truco: si no sale nada, no siempre es error. Puede que el filtro no coincida con los datos.

5) Recordatorio rapido (para clase)



<root>
  <item id="A1">texto</item>
</root>


//item            -> selecciona todos los item
//item[@id='A1']  -> filtra por atributo
count(//item)     -> cuenta nodos
//item[text()='texto'] -> filtra por texto exacto

Truco: primero prueba //libro. Luego filtra con [...].

6) Errores tipicos (lo que mas se ve en 1 DAM)

  • XML mal cerrado: falta </tag> o hay anidacion incorrecta.
  • Texto sin comillas en atributos: id=L1 esta mal, debe ser id="L1".
  • XPath no devuelve nada: no siempre es error, revisa el filtro.
  • XSD con sequence: el orden de elementos importa.
  • Keyref: referencias como S999 fallan si no existe ese id.
