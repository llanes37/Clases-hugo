# NIVEL 1 - SENCILLO

Este nivel es la puerta de entrada del bloque.

Su objetivo no es complicar al alumno, sino darle una estructura clara para
empezar bien cuando todavía confunde ideas básicas como:

- clase y objeto
- atributo y variable
- constructor y método
- modificar un objeto y mostrarlo por pantalla

---

## Qué se trabaja aquí

En este nivel se entrena sobre todo:

- crear una clase sencilla con pocos atributos
- hacer un constructor con parámetros
- implementar getters
- escribir métodos cortos que cambian el estado del objeto
- crear varios objetos en `main`
- probar llamadas a métodos y ver el cambio en pantalla

Es el nivel adecuado para alumnos que todavía necesitan repetir el patrón
fundamental varias veces antes de pasar a arrays de objetos.

---

## Cómo se debe usar este nivel

Orden recomendado de trabajo:

1. leer primero el enunciado
2. abrir después la guía explicada
3. resolver la plantilla `.java`
4. comparar al final con la solución comentada

La idea es que el alumno no copie desde el principio, sino que vea:

- qué pide el ejercicio
- por dónde conviene empezar
- qué partes son obligatorias
- cómo queda una solución limpia y entendible

---

## Ejercicio actual

- `PRODUCTO_BASICO`

Este ejercicio sirve como patrón inicial porque obliga a hacer justo lo
esencial:

- una clase simple
- varios objetos
- métodos de consulta
- métodos que modifican el estado

No mete todavía arrays ni estructuras más grandes. Eso permite centrar toda la
atención en entender bien la base.

---

## Qué contiene cada carpeta de ejercicio

Dentro de cada ejercicio de este nivel debe haber siempre estas cuatro piezas:

- archivo base `.java`
  Plantilla para trabajar en clase o en casa.
- guía `.md`
  Explica el sentido del ejercicio, el orden correcto y los errores típicos.
- enunciado `.md`
  Versión más parecida a examen real.
- archivo `_Solucion.java`
  Solución completa, comentada y pensada para corregir.

---

## Referencia para crecer este nivel

Cuando añadas más ejercicios a `NIVEL_1_SENCILLO`, conviene que sigan esta
misma idea:

- pocos atributos
- una sola clase principal
- métodos cortos y claros
- práctica de creación de objetos en `main`

Si un ejercicio ya necesita arrays de objetos, búsquedas o cálculos sobre varios
elementos, normalmente ya no pertenece a este nivel y debería pasar al
intermedio.
