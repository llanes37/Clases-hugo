# EJERCICIOS POR NIVELES - 1 DAM

> Bloque de repaso y preparacion de examen  
> Enfocado en los patrones que mas se repiten en Programacion de 1 DAM

---

## Idea del bloque

Este bloque no esta pensado para meter teoria nueva.

Su funcion es otra:

- consolidar lo importante
- entrenar patrones de examen
- reducir bloqueos del alumno
- dar una forma clara de empezar los ejercicios

Aqui se trabaja siempre sobre lo que mas suele caer:

- clases y objetos
- metodos
- arrays
- arrays de objetos

---

## Filosofia didactica

En muchos casos el alumno falla no porque no haya visto el contenido, sino
porque no sabe:

- por donde empezar
- en que orden hacer las partes
- como comprobar si va bien
- como pasar de una clase simple a un ejercicio tipo examen

Por eso este bloque esta organizado por niveles y cada ejercicio lleva:

- archivo `.java` con Better Comments y TODO guiados
- archivo `.md` con enunciado, orden de resolucion y errores tipicos

La idea es que el alumno aprenda un patron de trabajo estable.

---

## Estructura actual

### `NIVEL_1_SENCILLO`

Primer ejercicio de entrada.

Objetivo:

- crear una clase sencilla
- hacer constructor y getters
- crear objetos en `main`
- modificar su estado con metodos simples

Es el nivel ideal para alumnos que aun dudan entre:

- clase y objeto
- atributo y parametro
- metodo y constructor

### `NIVEL_2_INTERMEDIO`

Segundo escalon del bloque.

Objetivo:

- crear una clase
- guardar objetos en un array
- recorrer el array con metodos
- contar, buscar, calcular y mostrar

Este nivel ya se parece mucho al examen tipico de 1 DAM.

### `NIVEL_3_TIPO_EXAMEN`

Tercer escalon del bloque.

Objetivo:

- mantener el patron de objetos + array + metodos
- subir un poco la autonomia del alumno
- incluir metodos de modificacion sobre objetos encontrados

Este nivel ya esta muy orientado a simulacro y examen real.

### `NIVEL_4_EXAMEN_FINAL`

Cuarto escalon del bloque.

Objetivo:

- trabajar ejercicios largos
- combinar varias clases
- usar varias colecciones
- resolver un problema mas parecido a examen final completo

Ejercicios incluidos:

- `RED_SOCIAL`
- `ORDENADORES`

## Forma recomendada de uso en clase

### Opcion 1: repaso guiado

1. Leer el `.md` en clase.
2. Resolver el `.java` paso a paso.
3. Ir ejecutando tras cada bloque.
4. Corregir al final en comun.

### Opcion 2: simulacro corto

1. Dar solo el `.java`.
2. Dejar el `.md` como apoyo si se bloquean.
3. Corregir explicando el orden correcto de resolucion.

### Opcion 3: recuperacion

1. Empezar por el nivel sencillo.
2. Cuando salga bien, pasar al intermedio.
3. Repetir el patron con cambios pequeños de contexto.

---

## Regla de oro para el alumno

Nunca empieces un ejercicio escribiendo cosas al azar.

El orden normal de trabajo debe ser:

1. leer el enunciado
2. decidir si hay clase, array o ambas cosas
3. construir primero la clase
4. crear despues los objetos en `main`
5. hacer luego los metodos de proceso
6. probar el resultado

---

## Objetivo final

Que el alumno llegue al examen pensando:

> "Esto ya lo he hecho antes. Cambia el contexto, pero el patron es el mismo."

Esa es la meta real de este bloque.
